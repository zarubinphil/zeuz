# 0001 — Пересборка: схема Афины + апгрейд знаний

- Статус: принято
- Дата: 2026-06-19

## Контекст

zeuz (v1.1) — рабочая фабрика, но: (1) конституция отставала от свежего синтеза базы знаний «Оптимизация агентных систем»; (2) раскладка (`агенты/`, `конституция/`, Russian root-файлы) не следовала схеме `athena-os` и нарушала правило latin-kebab из `rules/structure.md` (NFD-баги в путях скриптов).

## Решение

1. **Латинский ренейм**: `агенты/`→`agents/`, `конституция/`→`rules/`, `BEST-PRACTICES.md`→`best-practices.md`.
2. **Схема Афины**: тонкий `CLAUDE.md` (Карта/Автозапуск/Команды/Конвенции/Не-делать), `specs/00-roadmap.md` (фазовый план, читать первым), `docs/decisions/` (ADR), `smoke/smoke.sh` (гейт), `README.md`+`README.ru.md`. Старые `CLAUDE_Зевс.md`/`Протокол_Зевс.md` свёрнуты в CLAUDE+specs.
3. **Апгрейд конституции** — добавлены/заострены: §0 harness>модель · §4 рой-экономика (×15 токенов, ×4.4 vs ×17.2) · §5 контекст=бюджет (73%/49%/10% cache) · §9 синтез-на-записи + retrieval-стек · §10 keep-or-revert + generator≠evaluator + circuit breakers · боевые правила (10).

## Границы

- **Рождаемые** системы сохраняют Femida-схему (`CLAUDE_<Имя>.md`, `Протокол_<Имя>.md`, `агенты/` Russian) — это их идентичность, не репо zeuz. Pipeline-промпты сборки не менялись по сути.
- `zeuz-pipeline.js`: обновлён только путь конституции (`rules/best-practices.md`). Логика ворот/стадий не тронута.

## Последствия

Нулевой риск регрессии pipeline (один путь). Схема теперь зеркалит Афину → проще навигация и онбординг. Конституция актуальна боевым логам.
