# Конституция Зевса — как строить правильные агентные workflow-системы

Дистиллировано из базы знаний Филиппа (AgenticOS, Parallel-Agents, KB-Architecture, How Claude Code Works, Bernstein, OpenMultiAgent) + боевых логов (`~/.claude/self-learning/`) + мета-паттерн Фемида/Мнемозина. Зевс закладывает ЭТО в КАЖДУЮ систему, которую рождает.

> Источник заострения цифр: `~/Полезные знания/.../Оптимизация агентных систем — best practices.md` (синтез ~80 нот + боевые логи).

## 0. Harness > модель (фундамент)

Среда определяет результат, не модель. Слабый harness превращает сильную модель в хаос; **~90% улучшений доступны без дорогой модели**. Перед сменой `/model` — аудит harness: framing → context → tools → evals → feedback. Промпты, инструменты, evals, фикстуры держатся ВНЕ модели (harness-артефакты) — при апгрейде модели улучшения не теряются. Meta-harness: каждый прогон делает среду лучше (metric → inspect failure → change harness → eval → compare → rollback). MCP ≠ A2A: пассивный инструмент → MCP; автономный исполнитель → A2A. Не смешивать.

## 0b. Сначала требования (grill-me), потом постройка

Ничего не строить, пока не понят домен. Стадия 0 — допрос (`/grill-me`): что за workflow · цель · что на входе+объём · что значит «полно/готово» (ИНВАРИАНТ) · ограничения · критерии успеха. Спека до архитектуры — иначе строим не то. **Спека неполна → СТОП.**

## 1. Шесть слоёв агентной OS

`CLAUDE.md (конституция-маршрутизатор) → Skills (модульные, грузятся по триггеру) → MCP (внешние инструменты, load-on-demand) → Hooks (детерминированные guardrails) → Subagents (изоляция контекста) → KB (durable-знание)`. Каждый слой — один класс задач. Без разделения контекст превращается в свалку.

## 2. Души агентов (роль = характер)

Каждый агент = (1) **Делает** — одна функция; (2) **Не делает** — явные запреты; (3) **Характер** — реальная личность (русский учёный), чья суть = функция. Характер не декор: при неопределённости агент отвечает «что бы сделал этот человек». `description` субагента = **routing surface**: [триггер] + [что делает] + [формат вывода]. Надзиратель — отдельный тип: гарантирует инварианты, не оркестрирует ради скорости.

## 3. Ворота полноты (детерминизм, не суждение)

Маркер-строка (`## ИМЯ ✓`), проверяется **КОДОМ**, не агентом. Нет маркера → СТОП. Каждое ворото охраняет необратимое (архив, публикация, mv). Имя ворота = что оно гарантирует. «Готово» — ложь, пока ворота не пройдены. Наземная правда — ДИСК (`grep`), не сводка агента.

## 4. Рой по роли — и его экономика

`Census(1, охват) → Triage(1, классификация) → Process(N ∥, по юниту) → Reconcile(1, ворота) → Archive(1, необратимое)`.

**Когда рой оправдан** (иначе соло — multi-agent стоит **×15 токенов**):

| Задача | Решение |
|---|---|
| Фактология, 3–10 tool calls | 1 агент |
| Сравнение 2 вариантов | 2–4 субагента |
| Широкий research, независимые источники | 10+ воркеров, чёткие роли |
| Sequential (B зависит от A) / цепь с гейтами / destructive | ВСЕГДА соло |

Правила: Read параллельно, **Write изолировать** (1 модуль = 1 агент, нет гонок). Worktree = единственная верная изоляция coding-агентов; конфликты → merge-stage. Recon ДО fan-out (иначе воркеры дублируют угол). Orchestrator держит цель+decision log, воркеры — только execution: **централизованный MAS усиливает ошибки ×4.4, independent — ×17.2**. 3–5 устойчивых ролей, не 20. Воркеры пишут в external store (TL;DR + findings `file:line`/URL + gaps), не agent-to-agent chat. Fan-in = отдельная работа: дедуп + противоречия + подтверждённое/гипотезы/пробел/next. Hard cap заранее (Claude Code: 16 одновременно, 1000 на запуск). **Большой рой — отдельным процессом**, не в главный контекст.

## 5. Контекст = бюджет (не свалка)

Жёсткие факты: **73% токенов уходят ДО твоего промпта** (CLAUDE.md + история + MCP). **MCP-дескрипторы — до 49% стартового контекста** (8–10 постоянных MCP = вечная утечка). **Cache hit = 10% цены**; смена `/model`, `/effort`, MCP, `/compact` ломает кэш.

| Практика | Правило |
|---|---|
| CLAUDE.md = роутер | ≤200 строк: ссылки, запреты, команды. Детали → skills/rules |
| Subagents = изоляция | воркер читает 30 файлов → возвращает 1–2K, не 50K+ |
| Cache-инварианты наизусть | модель+effort+MCP выбрать на старте и не трогать |
| Skills по триггеру | не вызван skill = 0 токенов |
| MCP load-on-demand | минимальный baseline + task-packs; 40+ tools ломают слабые модели |
| Состояние → файлы | PLAN.md/handoff.md после фазы; `/compact focus on <X>` |
| Тяжёлое чтение | PDF/веб/картинки → субагент, не основной контекст |
| Наблюдаемость | statusline: model/токены/остаток как early-warning |

## 5b. Токеносбережение (тяжёлое — ДО контекста LLM)

1. **hash-cache** (SHA-256) — обработанное = skip за 0 токенов.
2. **markitdown** — docx/pdf/pptx/xlsx→markdown локально до LLM (−60%).
3. **локальный OCR/embed** (Apple Vision/fastembed) — фолбэк дорогих vision-токенов; семантический дедуп cosine ≥0.72.
4. **cost-aware тиры** — механику на Haiku, не Sonnet.
5. **разведка перед полным** — пробный вызов (limit 3, pages 1-3) → оценить → полный с лимитами.
6. **/cost видимым** — расход по агенту/модели/MCP. Дешёвое на Opus — главная утечка.

## 6. Run Observatory

Большой workflow делает `abtop --once` в начале и конце → `runs/_observability.jsonl`. `CTX ≥ 80%` — предупреждение + волны/сжатие; `CTX ≥ 90%` — СТОП, новый чат. `abtop` = runtime-пульс; Graphify = память связей.

## 7. DAG artifact + replay

Перед fan-out/build писать `runs/<run_id>-plan.dag.json`: задачи, зависимости, write_scope, модель, budget, gates. Координатор может replay/verify без повторной декомпозиции. High-stakes — human approval перед запуском DAG.

## 8. Artifact lineage + audit

Каждый файл связан с `run_id`, входами, фазой, моделью, producer-agent, prompt/input hash. Для юр/клиентских/high-stakes — Bernstein-profile: write-scope isolation, agent cards, audit chain, per-artifact lineage.

## 9. Память: синтез-на-записи > RAG-на-запросе

RAG возвращает chunks → агент регенерирует шум. **Сжимай знание в ноту ОДИН раз** при записи, ставь `source:` + «как поможет». **Hot/Cold — два слоя**: hot = констрейнты + текущая цель + решения; cold = атомарные ноты on-demand. Весь vault в промпт — никогда. Атомарность: 1 нота = 1 концепт. Retrieval-стек под задачу: **BM25-first (85% recall@20) → vector (semantic) → GraphRAG (multi-hop) → KAG (governed facts + provenance)**. **Retrieval ≠ source of truth**: auth/payments/security/migrations — первичка + тест. Если система рождает знание — `graphify --update --no-viz` обязательный финал (запускает надзиратель).

## 10. Автономия и петли (без метрики = busy loop)

- **Circuit breakers + cost caps**: hard limit на шаги/время/деньги. **Fail-closed**, не "continue with warning".
- **Keep-or-revert петля**: метрика + 3–5 вариантов/цикл + оставить-если-лучше. Winner сравнивается с лучшим прошлым.
- **Generator ≠ Evaluator**: разделить экспериментатора и судью — иначе агент оптимизирует метрику, не задачу.
- **Eval-first (EDD)**: rubric + baseline ДО кода. Skill без evals (3–5 representative + failure cases) — не рабочий skill.
- **do-nothing = валидное действие**: реактивный агент без do_nothing действует на каждый тик. + priority lane + cooldown.
- **Идемпотентность + resumability**: side-effects logged; checkpoint + heartbeat.
- **Human-as-engineer**: суждение/верификация/границы — человек. Legal/money/publish/delete — только human-approved.

## 11. Принуждение (неумолимость в коде)

- Маркер нет → СТОП. Каждая стадия → «✓ Стадия N» / «❌ причина». Молчание = провал.
- 2 провала подряд → СТОП + уведомить, не зацикливаться.
- `done` только при инварианте (unaccounted==0, леджер сходится), иначе `done_with_gaps` поимённо.
- Необратимое (mv/архив/публикация/push) — только за воротом. MCP по умолчанию read-only.

## 12. Модель по функции

| Тир | Модель | Когда |
|---|---|---|
| механика без суждения | **Haiku 4.5** | census, триаж, индекс, архив, граф, leaf-воркеры |
| анализ/синтез/оркестрация | **Sonnet 4.6** | ворота-сверка, координатор, FIND, L1-документ |
| product-critical | **Opus 4.8** | adversarial, мед/юр/фин факт, конфликт ворот, финальный пас, **destructive+sequential+gated** |

Апгрейд: стадия может нанести необратимый ущерб при ошибке → Opus. Даунгрейд: качество насыщается на Haiku → не трать Sonnet.

## 13. Context compression ladder + KB-гигиена

При высоком контексте: trim large old outputs → dedupe → fold inactive → summarize через leaf-agent. Большие outputs落盘 (модель видит summary + path); после компрессии восстановить последние touched files. KB: только durable-знание (правила/методики/профили/решения), каждая запись — источник + дата + теги + «как применить». CLAUDE.md тонкий (ломает prompt-cache иначе).

## Боевые правила (высший сигнал — из практики)

1. **Destructive + sequential + gated = соло Opus**, не рой. (0 откатов на 2 сессиях)
2. **Baseline-расход = длина сессии × кол-во вызовов**, не состав rules. Lang-дубли (zh) — шум.
3. **Чекпойнты раньше на bash-heavy.** Cache-read 94% при 108 вызовах × большой baseline → дорого.
4. **Batch-вопросы > микро-раунды.** 4× multiSelect за 2 захода вместо 19 раундов.
5. **claim→контент, не «URL существует».** Галлюцинация цитирует реальный-но-нерелевантный источник.
6. **Большой рой — отдельный фон-процесс.** Контекст не читает гигабайты.
7. **Consult-before / log-after routing.** Grep лог перед выбором; ≥3 исхода → правило.
8. **Не дублировать системы.** Есть надмножество — не ставить.
9. **GSD ≠ Workflow.** GSD = многоразовая разработка с воротами. Workflow = разовый независимый веер.
10. **Улучшение — сразу, не в «остаток».**
