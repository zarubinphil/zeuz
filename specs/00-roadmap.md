## Что рождает фабрика (комплект новой системы домена D)

`CLAUDE_<Имя>.md` (конституция+автозапуск) · `Протокол_<Имя>.md` (SSOT) · `агенты/*.md` (души русских учёных по ролям) · `<имя>-pipeline.js` (workflow с воротами) · опц. инструменты (скрипты/кэши).

Каждая система ОБЯЗАНА (из `rules/best-practices.md`): души-личности · ворота полноты (код) · токеносбережение · рой по роли · Run Observatory (`abtop --once`) · DAG artifact · artifact lineage · graphify-финал (если рождает знание) · принуждение · модель по функции · harness>модель.

## Железные правила фабрики

1. **Спека неполна → СТОП.** Без grill-me не строить.
2. Каждая стадия → «✓/❌ + причина»; 2 провала → СТОП + уведомить.
3. Система не сдаётся без вердикта **ГОТОВА** Зворыкина.
4. Имена агентов — реальные русские учёные по роли, даты ВЕРИФИЦИРОВАНЫ, занятых не повторять.
5. Каждая система ОБЯЗАНА содержать инварианты конституции (см. выше).
6. Файлы: `~/Проекты/zeuz/` (проект) · активный workflow `~/.claude/workflows/zeuz-pipeline.js`.
